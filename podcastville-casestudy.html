프로젝트 개요

Project Overview

하나의 팀이자, 하나의 프로덕트

One person. One product.

PodcastVille은 팟캐스트 청취 경험을 개선하기 위해 기획된 서비스입니다. 기존 버전은 시각적 일관성, 사용성, 퍼포먼스 모두에서 개선이 필요한 상태였습니다. 이번 리뉴얼에서는 Apple Podcasts 수준의 인터랙션과 디자인 품질을 목표로, 설계부터 코드 배포까지 모든 단계를 직접 담당했습니다.

PodcastVille is a podcast listening platform that needed a complete overhaul — in visual consistency, usability, and performance. This renewal project targeted Apple Podcasts-level interaction quality, with me handling every stage from concept to deployed code.

과제와 목표

Challenge & Goals

무엇이 문제였고, 무엇을 만들었나

What was broken, and what we built

🔴 기존 문제점

🔴 Previous Issues

  • 시각적 계층 구조가 불분명하고 브랜드 일관성 부재
  • 모바일 반응형 레이아웃 미흡 (375px~1440px)
  • 클릭 불가 카드, 인터랙션 피드백 없음
  • 플레이어 UI가 현대적 스트리밍 서비스 수준에 미달
  • 성능 최적화 미흡 (이미지, 폰트, 렌더링)
  • Unclear visual hierarchy and lack of brand consistency
  • Poor responsive layout across 375px–1440px
  • Unclickable cards, no interaction feedback
  • Player UI below modern streaming service standards
  • Insufficient performance optimization (images, fonts, rendering)

🟣 리뉴얼 목표

🟣 Renewal Goals

  • Apple Podcasts 수준의 인터랙션과 시각적 품질 달성
  • 완전한 반응형 — 모바일·태블릿·데스크탑 대응
  • 모든 카드·컨트롤이 작동하는 기능 완성도
  • 풀스크린 플레이어, 앰비언트 애니메이션 구현
  • Next.js App Router + SCSS Modules 기반 클린 아키텍처
  • Achieve Apple Podcasts–level interaction and visual quality
  • Fully responsive — mobile, tablet, and desktop
  • All cards and controls fully functional
  • Full-screen player with ambient canvas animation
  • Clean architecture with Next.js App Router + SCSS Modules

작업 프로세스

Process

기획부터 배포까지

From brief to deployment

디자이너와 개발자 역할을 오가며, 각 단계에서 품질 기준을 직접 정의하고 구현했습니다.

Switching between designer and developer modes, I defined quality standards at each stage and implemented them myself.

01
리서치 & 분석
Research & Analysis
Apple Podcasts, Spotify, YouTube Music 벤치마킹. 기존 서비스 사용성 감사 및 개선 포인트 도출.
Benchmarked Apple Podcasts, Spotify, and YouTube Music. Conducted usability audit of existing service.
02
UX 설계
UX Design
정보 아키텍처 재구성, 내비게이션 플로우 설계, 와이어프레임 작성 및 프로토타이핑.
Restructured information architecture, designed navigation flow, created wireframes and prototypes.
03
UI 디자인
UI Design
다크 테마 디자인 시스템 구축. 컬러, 타이포그래피, 컴포넌트 토큰 정의.
Built dark-theme design system. Defined color, typography, and component tokens.
04
개발 & 구현
Development
Next.js 15 + TypeScript + SCSS Modules로 전체 컴포넌트 구현. 반응형 및 접근성 대응.
Built all components in Next.js 15 + TypeScript + SCSS Modules. Handled responsive design and accessibility.
05
QA & 최적화
QA & Optimization
멀티 디바이스 반응형 검증, 인터랙션 버그 수정, 이미지 최적화.
Multi-device responsive validation, interaction bug fixes, image optimization.
06
배포
Deployment
GitHub 연동 Vercel CI/CD 파이프라인 구성. 프로덕션 배포 완료.
Set up GitHub-linked Vercel CI/CD pipeline. Deployed to production.

기술 스택

Tech Stack

현대적 프론트엔드 아키텍처

Modern front-end architecture

성능, 유지보수성, 확장성을 모두 고려해 기술 스택을 선정했습니다. Tailwind를 제거하고 SCSS Modules로 전환해 디자인 토큰을 완전히 통제했습니다.

The stack was chosen for performance, maintainability, and scalability. Tailwind was removed in favor of SCSS Modules to gain full control over design tokens.

주요 기능

Key Features

Designed and built by hand

🎨

앰비언트 풀스크린 플레이어

Ambient Full-Screen Player

Canvas API + requestAnimationFrame으로 앨범 아트워크를 4개의 회전 스프라이트로 분산 렌더링. 현재 트랙에 맞게 배경이 동적으로 변화합니다.

Canvas API + requestAnimationFrame renders album art as 4 rotating sprites for a living ambient background that dynamically updates with the current track.

📐

CSS Container Queries 반응형

CSS Container Query Layout

미디어 쿼리 대신 Container Queries를 활용해 카드 그리드가 부모 컨테이너 크기에 반응합니다. 재사용성과 예측 가능성이 크게 향상됩니다.

Cards respond to parent container size using CSS Container Queries instead of media queries — dramatically improving reusability and predictability.

🎙️

팟캐스트 디테일 페이지

Podcast Detail Page

에피소드 목록, 평점 차트, 호스트 프로필, 관련 팟캐스트 추천 등 Apple Podcasts 수준의 상세 페이지를 완전 구현했습니다.

Full-featured detail page with episode list, star-rating chart, host profiles, and related podcast recommendations — matching Apple Podcasts quality.

🔍

실시간 검색

Real-time Search

Shows, Episodes, Channels, Categories를 동시에 검색하는 실시간 필터링. 결과가 즉각적으로 업데이트됩니다.

Real-time filtering across Shows, Episodes, Channels, and Categories simultaneously — results update instantly as you type.

📊

차트 & 탭 필터

Charts & Tab Filters

8개 카테고리 탭으로 Top Charts를 필터링. 상태 관리 없이 URL 기반 라우팅으로 구현해 SEO와 공유 가능성을 보장합니다.

Filter Top Charts by 8 categories via tab UI. Implemented with URL-based routing instead of state — ensuring SEO and shareability.

📱

완전 반응형 레이아웃

Fully Responsive Layout

375px 모바일부터 1440px 데스크탑까지 완벽 대응. MobileNav, 사이드바 전환, 플레이어바 레이아웃 변화를 세밀하게 구현했습니다.

Seamless from 375px mobile to 1440px desktop. MobileNav, sidebar toggle, and player bar layout changes all implemented with precision.

디자인 시스템

Design System

일관성의 기반

The foundation of consistency

모든 컴포넌트의 기반이 되는 디자인 토큰을 CSS 변수로 정의했습니다. 색상, 타이포그래피, 간격, 반경 모두 하나의 소스에서 관리됩니다.

All components are built on design tokens defined as CSS custom properties. Color, typography, spacing, and radius are all managed from a single source.
